# 임무: KAYLA 랜딩페이지 A - PDF 대조 QA 및 수정 루프

## 배경
jensen(다른 에이전트)이 1차 빌드를 완료해 배포했다. 너(openclaw)는 이 배포본을 원본 PDF와 픽셀 단위로 계속 대조하며 다른 점을 찾아서 그 자리에서 고치고 재배포하는 역할이다. 사용자(Kayla)는 약 2.5시간 뒤 복귀하며, 그때까지 최대한 PDF와 동일하게 다듬어 놓아야 한다.

## 필수 요구사항 (절대 준수)
1. 색상/폰트/레이아웃/카피가 PDF 원본과 동일해야 한다.
2. 실제 사진 7장이 전부 올바른 위치에 있어야 한다 (텍스트/일러스트 대체 금지).
3. 클릭 유도 요소(신청하기, CTA 버튼 등) 전부 `https://litt.ly/kayla.artlab/sale/g2Gbjpb` 로 연결되어야 한다 — 예외 없음.
4. 랜딩페이지(index.html) 전체에서 컬러 변수와 폰트가 하나로 통일되어 있어야 한다.

## 작업 방식 (반복)
1. PDF 페이지를 섹션별로 크롭해서(`landingA_page_1.png`를 y좌표 구간별로 잘라서) 색상 hex값, 폰트, 텍스트, 레이아웃을 확인한다.
2. 배포된 실제 URL을 브라우저로 열어서 같은 섹션과 비교한다.
3. 다른 부분(색상 틀림, 텍스트 다름, 이미지 위치/크기 다름, 버튼 링크 오류, 반응형 깨짐 등)을 찾으면 index.html을 직접 수정한다.
4. `git add -A && git commit -m "..." && git push`로 재배포한다.
5. 60~90초 대기 후 다시 curl로 200 확인하고, 다시 1번부터 반복한다.
6. 이 루프를 최소 5회 이상, 시간이 허락하는 한 계속 반복해서 최대한 PDF와 가깝게 만든다.

## 매 루프마다 반드시 확인
- `grep -o 'litt.ly[^"]*' index.html` 결과 — 모든 CTA가 결제링크로 연결되어 있는지
- `grep -oE '<img [^>]*src="[^"]*"' index.html` 결과 — 이미지 누락 없는지
- 실제 배포 URL이 HTTP 200인지 (`curl -s -o /dev/null -w "%{http_code}" https://kaylaartlab-commits.github.io/kayla-art-lab-preview/`)

## 주의사항
- Windows, git-bash(MSYS) 쉘. 경로 `/c/Users/kayla/...` 형태 사용 가능.
- Python 3.11 (PIL, PyMuPDF 설치되어 있음).
- shared-memory 저장소는 건드리지 말 것. kayla-art-lab-deploy 저장소만 사용.
- 지어내지 말고, 실제로 확인한 것만 보고할 것.
- 작업이 끝나거나 중단되면, 무엇을 몇 번 고쳤는지, 최종 배포 URL, 남은 이슈가 있다면 무엇인지 요약해서 보고할 것.
